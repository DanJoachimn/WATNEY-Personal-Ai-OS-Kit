# Affiliate links — single source of truth

> Every affiliate / referral link the kit surfaces lives **here, once.** The install playbook and docs reference these by name, so swapping a link (or adding a program) is a **one-file edit** — never a hunt across the repo.
>
> **Status: LIVE.** Real affiliate links are in place. If a slot is ever blank, the kit falls back to the plain link — which still works perfectly for the user, it just earns no kickback.

## The rule (how the kit uses these) — a nudge, never a sale

- The installing AI surfaces the relevant link at the **natural moment** — voice-in setup → Wispr Flow; voice-out setup → ElevenLabs; meeting capture → Granola; the newsletter plug → Beehiiv. Never a hard pitch, never gated. A recommendation from a friend, offered once, in passing.
- **The user always gets the free tier first.** They only ever upgrade if *they* decide the tool is worth it — at which point the affiliate credit applies. **The user never pays a cent extra.**
- The **disclosure line (below) is shown the first time an affiliate link appears in a session.** Transparent by design — that honesty is the whole positioning.

### The natural upgrade story per tool (why the free tier converts on its own)

- **Wispr Flow** — free tier is **2,000 words/week**. People get hooked on talking instead of typing; the efficiency sells the upgrade by itself. Surface it early so they dictate the rest of the install.
- **ElevenLabs** — free tier gives a handful of genuinely good voices (enough for the aha-moment). The **large voice library + more speaking time** is the paid plan. Mention the library exists; let curiosity do the rest.
- **Granola** — free for the first stretch of meetings, then the "every call auto-captured to my vault" habit is what converts. Same shape as Wispr: the habit sells it.

### Beehiiv — special placement rule (end only)

Beehiiv is **not** pitched as a tool the user installs. It appears in exactly one place: under the **All Gravy Times** newsletter plug at the very end of the install (and in the README's newsletter section), as a soft credit line:

> *Presented with [Beehiiv](https://www.beehiiv.com?via=daniel-joachim-nielsen).*

Just that — "Beehiiv" hyperlinked to the affiliate link. No pitch. It reads as "here's what powers the newsletter," and quietly plants the platform for anyone who later wants to start their own.
