# People/

One file per real person in [PARTNER_NAME]'s working life. Clients, suppliers, collaborators, podcast guests, key relationships.

## Why this matters for the AI

When [PARTNER_NAME] says *"draft a follow-up to Sarah,"* the AI reads `People/Sarah [Lastname].md` first. It now knows: who Sarah is, the tone she expects, what they last discussed, what's outstanding. The draft lands on first try instead of asking 5 clarifying questions.

## How to add a person

1. Copy `_Person template.md` and rename to the person's name (e.g. `Sarah Chen.md`).
2. Fill in the sections.
3. Tell the AI: *"new person — [name] — read their file."*

## What the AI does with these files

- **Read** when the person is mentioned in a request
- **Append** to the "Recent context" and "Threads I'm tracking" sections after meetings or substantive interactions
- **Never** delete or archive without explicit ask

## Notability gate

Don't create a People/ entry for someone mentioned once. Wait until they're mentioned twice, OR once with substantive content (e.g. they're now actively a client, supplier, or collaborator). Avoids file clutter.
